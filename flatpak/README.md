# Flathub Submission

This directory contains the files needed to publish PDFApps to [Flathub](https://flathub.org/).

## Files

- `io.github.nelsonduarte.PDFApps.yml` — Flatpak manifest
- `python-modules.yml` — Python dependencies (auto-generated by `req2flatpak`)
- `io.github.nelsonduarte.PDFApps.metainfo.xml` — AppStream metadata
- `io.github.nelsonduarte.PDFApps.desktop` — desktop file
- `pdfapps.sh` — launcher script
- `requirements-pinned.txt` — pinned versions used to generate `python-modules.yml`

## Validated

- AppStream metadata: `appstreamcli validate` passed
- Desktop file: `desktop-file-validate` passed
- Manifest YAML: parses without errors
- Python modules YAML: 8 sources with real SHA256 hashes (generated by req2flatpak)

## Local testing

```bash
# Install Flatpak builder (Linux only)
sudo apt install flatpak-builder

# Add Flathub remote
flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo

# Install KDE runtime
flatpak install flathub org.kde.Platform//6.7 org.kde.Sdk//6.7

# Build (from project root)
flatpak-builder --force-clean build-dir flatpak/io.github.nelsonduarte.PDFApps.yml

# Run locally
flatpak-builder --run build-dir flatpak/io.github.nelsonduarte.PDFApps.yml pdfapps
```

## Re-generating Python dependencies

If you update `requirements-pinned.txt`, regenerate `python-modules.yml`:

```bash
pip install req2flatpak pyyaml
python -m req2flatpak \
  --requirements-file flatpak/requirements-pinned.txt \
  --target-platforms 312-x86_64 \
  --yaml \
  --outfile flatpak/python-modules.yml
```

## Submission to Flathub

1. Fork [flathub/flathub](https://github.com/flathub/flathub) on GitHub
2. Create a new branch named `new-pr`
3. Copy all files from this `flatpak/` directory into the root of your fork
4. Open a PR following the [submission guide](https://docs.flathub.org/docs/for-app-authors/submission)
5. Wait for review (typically a few weeks)

## Notes

- **Auto-updater is disabled inside Flatpak** (detected via `FLATPAK_ID` env var) — Flatpak handles updates itself
- **Tesseract and Ghostscript are NOT bundled** — users would need to install them separately or rely on system packages. OCR and advanced compression will not work in the Flatpak by default.
- **PySide6 binds to Qt6** provided by `org.kde.Platform//6.7` runtime
- **App data**: stored in `~/.var/app/io.github.nelsonduarte.PDFApps/` instead of `~/.pdfapps_config.json` (Flatpak sandboxing)
- **File access**: `--filesystem=home` allows the user to open PDFs anywhere in their home directory
